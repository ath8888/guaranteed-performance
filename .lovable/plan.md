## What I'll add â€” offline support via `vite-plugin-pwa`

Per the Lovable PWA skill, hand-written service workers are forbidden; offline must be generated by `vite-plugin-pwa` with strict preview safety.

1. **Install** `vite-plugin-pwa` as a dev dependency.
2. **Configure it in `vite.config.ts`** with:
   - `registerType: "autoUpdate"`, `injectRegister: null`, `devOptions: { enabled: false }`
   - `strategies: "generateSW"` emitting `/sw.js`
   - `NetworkFirst` for HTML navigations, `CacheFirst` only for same-origin hashed assets
   - Navigation fallback excludes `/~oauth`
   - Point at the existing `public/manifest.webmanifest` (don't duplicate manifest fields)
3. **Add a single guarded registration wrapper** (`src/lib/pwa-register.ts`) that refuses to register when:
   - `!import.meta.env.PROD`
   - inside an iframe
   - hostname starts with `id-preview--` or `preview--`
   - hostname is/ends with `lovableproject.com`, `lovableproject-dev.com`, or `beta.lovable.dev`
   - URL has `?sw=off` (kill switch)
   In any refused context, the wrapper unregisters any existing `/sw.js` registration.
4. **Call the wrapper once** from `RootComponent` inside a `useEffect`.
5. **Publish** so the latest preview code (including everything from prior turns) goes live on `clear-standard-path.lovable.app`, where Android can install it as a PWA.
